# 服装多场景 Prompt MVP

服装模式与鞋履模式共用同一个页面和 Prompt 管线，通过 `ProductContext` 选择 `GarmentProductAdapter`。默认仍为鞋履模式；服装模式隐藏鞋款与生成图片预留区，改为服装品类、服装信息和本地参考图。

参考图仅使用浏览器 `Object URL` 预览，支持 PNG、JPEG、WebP，最多 5 张，至少 4 张且必须包含“正面完整图”才允许生成正式 Prompt。图片不会上传、不会写入 Prompt，也不会发送网络请求。用户复制英文 Prompt 后，需将同一组参考图手动上传至 Image 2.0。

服装适配器把上传服装作为固定主产品，保护轮廓、肩部、领口、袖型、腰线、长度、门襟、口袋、面料、颜色、纹理、垂坠与工艺细节；动作只允许自然褶皱、垂坠和穿着压缩。辅助搭配只能围绕主服装生成，不能替换、覆盖、改色或重新设计主服装。

服装场景继续复用现有 `TeamScenePreference`、场景光线、城市、相机、动作、人物、季节和真实感系统。`garmentSceneCompatibility` 只做品类与图片类型过滤，不复制场景文案。普通服装可覆盖日常、通勤、社交、文化、购物、酒店和旅行；婚纱、礼服、轻运动服按品类排除明显冲突场景。

服装图片类型使用现有内部值，但前端显示“模特穿着图、对镜试穿图、生活街拍图、品牌氛围图、服装细节 / 拍摄花絮、服装平铺 / 静物图”。组图继续复用现有主题和复制能力；棚内组图数量沿用现有 5/8 入口，普通主题沿用 3/5 入口。鞋履模式的主题、鞋型保护和棚拍逻辑保持兼容。

最终 Prompt 经过服装模式专属净化，避免鞋履主产品保护词污染服装 Product 段。后续若接入 API，应在当前本地参考图和 Prompt 复制流程之外新增独立后端边界，不把 API Key 放进前端。
